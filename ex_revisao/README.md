📚 Exercícios de POO — Relacionamentos entre Classes

Este conjunto de exercícios aborda os principais tipos de relacionamento em Programação Orientada a Objetos: associação, agregação e composição.

🧠 Parte 1: Classe de Associação
🩺 Exercício 1: Sistema de Clínica Médica

Modela o relacionamento entre médico e paciente através de uma consulta.
A classe Consulta representa a associação e armazena informações próprias, como data, valor e os objetos envolvidos.

📚 Exercício 2: Sistema de Biblioteca

Representa o empréstimo de livros entre leitor e livro.
A classe Emprestimo funciona como associação, contendo datas e o vínculo entre os dois objetos, além de exibir essas informações.

🧩 Parte 2: Agregação
🏀 Exercício 3: Time de Basquete

Demonstra um relacionamento onde um time possui vários atletas, mas os atletas existem independentemente do time.
Mesmo que o time deixe de existir, os atletas continuam existindo.

💻 Exercício 4: Projeto de Software

Mostra um projeto que reúne vários programadores.
Os programadores não dependem do projeto para existir, caracterizando uma relação de agregação.

🏗️ Parte 3: Composição
🖥️ Exercício 5: Computador e Componentes

Representa um computador que possui um processador como parte essencial.
O processador só existe dentro do computador, caracterizando uma relação de composição.

🏢 Exercício 6: Edifício e Apartamentos

Mostra um edifício que contém apartamentos criados internamente.
Os apartamentos fazem parte do edifício e não existem de forma independente.

🔥 Parte 4: Agregação + Composição
🛒 Exercício 7: Sistema de Vendas

Combina agregação e composição em um mesmo sistema.
Produtos e clientes existem de forma independente, enquanto os itens de venda só existem dentro de uma venda.

🎬 Exercício 8: Sistema de Cine-Teatro

Modela uma sessão de cinema que se relaciona com um filme (agregação) e com ingressos (composição).
O filme existe independentemente, enquanto os ingressos só existem dentro da sessão.
